# 🎼 melody-physics-lab

> Гибридный пайплайн **«сепарация + транскрипция»** для перевода аудио в MIDI — и исследовательская площадка, чтобы понять **физику (природу) мелодии**.

## Зачем

Существующие end-to-end модели (например, [magenta/mt3](https://github.com/magenta/mt3)) пока не дают точной транскрипции плотной многоголосной музыки. **Поэтому гибридный пайплайн «сепарация + транскрипция» сейчас даёт наилучший практический результат:**

1. **Сепарация** — разделяем микс на инструментальные дорожки (stems).
2. **Транскрипция** — переводим каждую дорожку в MIDI отдельной специализированной моделью.
3. **Очистка** — квантизация, исправление, экспорт нот.

## Большая цель (мечта)

Понять **природу мелодии**:

- Почему сочетание звуковой гармонии вызывает у людей **озноб от головы до пят** (*frisson*)?
- Что делает мелодию **запоминающейся** — так, что её можно воспроизвести по памяти после **первого** прослушивания?

MIDI-транскрипция здесь — это инструмент: она превращает звук в структурированные данные (ноты, интервалы, ритм, гармония), которые можно анализировать численно.

## Структура репозитория

```
melody-physics-lab/
├── README.md              ← вы здесь
├── requirements.txt       ← зависимости пайплайна
├── docs/
│   ├── pipeline.md        ← детальный гибридный пайплайн + модели
│   └── research-roadmap.md← план исследования frisson и запоминаемости
├── src/                   ← код пайплайна (заполняется)
├── data/                  ← аудио и stems (не коммитим)
└── experiments/           ← ноутбуки и эксперименты
```

## Быстрый старт (план)

```bash
pip install -r requirements.txt
# 1. Сепарация
python -m demucs --two-stems=vocals path/to/track.wav
# 2. Транскрипция stems в MIDI (см. docs/pipeline.md)
# 3. Анализ MIDI (см. experiments/)
```
